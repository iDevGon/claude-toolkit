# Session Save — 세션 중간 저장

현재 세션에서 지금까지 수행한 작업을 요약하여 `~/.claude/memory/<티켓>/<레포>/` 에 저장한다.
세션 종료를 기다리지 않고 원하는 시점에 작업 기록을 남길 수 있다.

## 사용법

```
/session-save
```

## 실행 절차

### 1단계: 티켓 번호 + 레포명 파싱

```bash
git rev-parse --abbrev-ref HEAD
basename "$(git rev-parse --show-toplevel)"
```

- 정규식: `^([A-Z][A-Z0-9]+-[0-9]+)` 패턴을 매칭한다.
- 티켓 번호를 파싱할 수 없는 경우 사용자에게 직접 입력받는다.

### 2단계: 현재 세션 작업 요약 생성

현재 대화의 맥락을 바탕으로 아래 구조의 요약을 작성한다.

```markdown
# 세션 메모리 — <작업 목표 한줄 요약>

- 일시: <`date +"%Y-%m-%d %H:%M:%S"` 명령 결과>
- 유형: 중간 저장

## 작업 목표
<이번 세션에서 달성하려는 것>

## 수행한 작업
- <변경 1>
- <변경 2>
- ...

## 주요 결정사항
- <결정>: <이유>
- ...

## 미완료 작업 / 다음 할 일
- [ ] <할 일>
- ...

## 참고사항
- <참고>
- ...
```

### 3단계: 파일 저장

```bash
date +"%Y-%m-%d-%H_%M_%S"
```

위 명령의 결과를 타임스탬프로 사용하여 저장한다.

```
~/.claude/memory/<티켓>/<레포명>/<timestamp>.md
```

디렉토리가 없으면 생성한다.

### 4단계: 결과 보고

```
세션 메모리 저장 완료: ~/.claude/memory/<티켓>/<레포명>/<timestamp>.md
```

## 제약사항

1. **현재 대화의 맥락만으로 요약을 작성한다.** 외부 MCP 호출이나 파일 조회 없이 현재까지의 대화 내용을 기반으로 한다.
2. **요약은 사실에 기반해야 한다.** 실제로 수행한 작업만 기록하고, 추측이나 계획은 "미완료 작업"에 넣는다.
3. **타임스탬프는 반드시 `date` 명령 결과를 사용한다.** (KST 보장)
