# Session Start — 작업 컨텍스트 로드

현재 브랜치에서 Jira 티켓 번호를 파싱하고, 티켓 내용 + 연결된 Notion 문서 + 이전 세션 기록을 한 번에 로드한다.

## 메모리 디렉토리 구조

```
~/.claude/memory/
└── <티켓번호>/
    ├── context.md                        # Jira + Notion 컨텍스트 (레포 공통)
    ├── <레포명>/
    │   ├── <timestamp>.md                # 세션 요약
    │   └── <timestamp>-raw.jsonl         # 세션 원본 (요약 전)
    └── <다른레포명>/
        └── ...
```

## 실행 절차

### 1단계: Jira 티켓 번호 + 레포명 파싱

```bash
git rev-parse --abbrev-ref HEAD
basename "$(git rev-parse --show-toplevel)"
```

**브랜치 컨벤션**: `PROJ-123/branch-name` 형태에서 `/` 앞 부분이 티켓 번호이다.

- 정규식: 브랜치 이름에서 `^([A-Z][A-Z0-9]+-[0-9]+)` 패턴을 매칭한다.
- **티켓 번호를 파싱할 수 없는 경우**: 사용자에게 직접 티켓 번호를 입력받는다.
  ```
  현재 브랜치에서 Jira 티켓 번호를 찾을 수 없습니다.
  작업 중인 Jira 티켓 번호를 입력해주세요 (예: PROJ-123):
  ```

출력:
```
티켓: PROJ-123 | 레포: repo-name
```

### 2단계: 컨텍스트 로드 (병렬 실행)

아래 작업을 **모두 병렬로** 실행한다.

#### 2-1. Jira + Notion 컨텍스트

캐시 경로: `~/.claude/memory/<티켓>/context.md`

**캐시 파일이 존재하면:**
- 해당 파일을 읽어서 컨텍스트로 사용한다. MCP 호출하지 않는다.

**캐시 파일이 없으면:**

A) 설정 파일 `~/.claude/config/session-pipeline.json`을 읽어서 `jiraCloudId`를 가져온다.
설정 파일이 없으면 사용자에게 Jira Cloud ID (예: `your-org.atlassian.net`)를 질문하고, 답변을 설정 파일로 저장한다.

```json
// ~/.claude/config/session-pipeline.json
{
  "jiraCloudId": "<사용자 설정값>"
}
```

`mcp__atlassian__getJiraIssue` 로 티켓 정보를 조회한다.

```
cloudId: <설정 파일의 jiraCloudId>
issueKey: <티켓 번호>
```

조회 후 다음을 정리한다:
- 제목 (summary)
- 상태 (status)
- 담당자 (assignee)
- 설명 (description) 요약
- 하위 이슈 / 연결된 이슈
- **description 또는 코멘트에 포함된 Notion URL** (`notion.so` 또는 `notion.site` 도메인) 수집

B) Notion URL이 발견되면 `mcp__notion__notion-fetch` 로 각 문서를 **병렬 조회**한다.

C) 결과를 캐시 파일로 저장한다. 캐시 일시는 반드시 `date +"%Y-%m-%d %H:%M:%S"` 명령을 실행한 결과를 사용한다 (KST 보장).

```
파일: ~/.claude/memory/<티켓>/context.md
```

캐시 파일 형식:
```markdown
# 티켓 컨텍스트: <티켓번호>

- 캐시 일시: <date 명령 결과>

## Jira 티켓
- 제목: <summary>
- 상태: <status>
- 담당자: <assignee>
- 설명: <description 요약>
- 연결된 이슈: <목록>

## Notion 문서
### <문서 제목 1>
<핵심 내용 요약>
```

#### 2-2. 이전 세션 기록 읽기

대상 경로: `~/.claude/memory/<티켓>/<레포명>/`

**A. 요약 파일 (`*.md`)**: 직접 읽어서 컨텍스트로 사용한다.

**B. Raw transcript 파일 (`*-raw.jsonl`)**: SessionEnd 훅이 자동 저장한 세션 원본.
  - **Agent 도구**를 사용하여 별도 에이전트에게 요약을 위임한다.
  - 에이전트 프롬프트:
    ```
    아래 JSONL 파일은 이전 Claude Code 세션의 대화 기록이다.
    파일 경로: <raw.jsonl 파일 경로>

    이 파일을 읽고 다음 구조로 요약해라:
    1. 작업 목표: 세션에서 달성하려 한 것
    2. 수행한 작업: 실제로 한 일 (변경한 파일, 구현한 기능, 수정한 버그 등)
    3. 주요 결정사항: 기술적 판단, 설계 결정
    4. 미완료 작업 / 다음 할 일
    5. 참고사항

    요약을 마크다운으로 작성하고, 원본 raw.jsonl 파일은 삭제해라.
    요약 결과를 같은 디렉토리에 <원본파일명에서 -raw.jsonl을 .md로 교체>로 저장해라.
    예: 2026-04-02-14_30_15-raw.jsonl → 2026-04-02-14_30_15.md

    요약 파일 형식:
    # 세션 메모리 — <작업 목표 한줄 요약>

    - 일시: <JSONL에서 추출한 세션 시작 시각>

    ## 작업 목표
    <내용>

    ## 수행한 작업
    - <변경 1>
    - ...

    ## 주요 결정사항
    - <결정>: <이유>
    - ...

    ## 미완료 작업 / 다음 할 일
    - [ ] <할 일>
    - ...

    ## 참고사항
    - <참고>
    - ...
    ```
  - **여러 raw 파일이 있으면 각각 별도 에이전트로 병렬 처리**한다.
  - 에이전트 완료 후 생성된 요약 `.md` 파일을 읽는다.

파일이 없으면 "이전 세션 기록 없음"으로 표시한다.

### 3단계: 컨텍스트 요약 보고

```
## 작업 컨텍스트 로드 완료

### Jira 티켓: PROJ-123
- 제목: <summary>
- 상태: <status>
- 담당자: <assignee>
- 설명 요약: <description 요약>
- (캐시 로드 / MCP 새로 조회)

### Notion 문서
- <문서 제목>: <핵심 내용 요약>
- (없으면 "연결된 Notion 문서 없음")

### 이전 세션 기록 (repo-name)
- <세션 날짜>: <세션 요약 핵심>
- (없으면 "이전 세션 기록 없음")

---
컨텍스트 로드가 완료되었습니다. 작업을 시작하세요.
```

## 제약사항

1. **티켓 번호를 찾지 못하면 반드시 사용자에게 질문한다.** 임의로 추측하지 않는다.
2. **병렬 실행 가능한 작업은 반드시 병렬로 실행한다.**
3. **Notion URL은 Jira 티켓의 description과 코멘트에서만 추출한다.**
4. **raw.jsonl 파일은 요약 완료 후 반드시 삭제하고 .md 요약 파일로 대체한다.**
5. **캐시 파일(`context.md`)이 있으면 MCP를 호출하지 않는다.** 캐시 갱신이 필요하면 `/session-refresh`를 사용한다.
6. **세션 기록은 현재 레포의 디렉토리만 읽는다.** 다른 레포 세션은 로드하지 않는다.
