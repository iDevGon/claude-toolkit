# Session Status — 메모리 현황 조회

`~/.claude/memory/` 에 저장된 티켓별 세션 메모리를 테이블 형태로 정리하여 보여준다.

## 사용법

```
/session-status              # 현재 브랜치 티켓 기준 상세
/session-status PROJ-123     # 특정 티켓 상세
/session-status all          # 전체 티켓 현황
```

## 실행 절차

### 1단계: 티켓 번호 결정

**파라미터가 `all`인 경우**: 전체 현황 모드로 진행 (2단계-A).

**파라미터로 티켓 번호가 주어진 경우**: 해당 티켓으로 진행 (2단계-B).

**파라미터가 없는 경우**: 현재 브랜치에서 티켓 번호를 파싱한다.

```bash
git rev-parse --abbrev-ref HEAD
```

- 정규식: `^([A-Z][A-Z0-9]+-[0-9]+)` 패턴을 매칭한다.
- 파싱 성공 시 해당 티켓으로 상세 모드 진행 (2단계-B).
- 파싱 실패 시 전체 현황 모드로 진행 (2단계-A).

### 2단계: 현황 출력

#### A. 전체 현황 (`all` 또는 브랜치 파싱 실패)

`~/.claude/memory/` 디렉토리가 없거나 비어있으면:
```
세션 메모리가 없습니다.
```

각 티켓 디렉토리를 순회하며 정보를 수집한다.

- 티켓 번호: 디렉토리명
- 레포 목록: 하위 디렉토리명 (context.md 제외)
- 컨텍스트 캐시: `context.md` 존재 여부
- 세션별 제목: 각 `.md` 파일의 첫 번째 `# ` 줄에서 `세션 메모리 — ` 이후 부분을 추출
- `*-raw.jsonl` 파일은 제목을 "*(미정리)*"로 표시

출력 형식:

```
## 세션 메모리 현황

### PROJ-123 (캐시: ✓)

| # | 레포 | 일시 | 제목 | 상태 |
|---|------|------|------|------|
| 1 | repo-a | 2026-04-03 16:15 | master 머지 충돌 해결 | 요약 완료 |
| 2 | repo-a | 2026-04-06 09:43 | 포인트 할인 UI 구현 | 요약 완료 |
| 3 | repo-b | 2026-04-06 11:00 | — | 미정리 (raw) |

### PROJ-456 (캐시: ✗)

| # | 레포 | 일시 | 제목 | 상태 |
|---|------|------|------|------|
| 1 | repo-a | 2026-04-04 10:20 | API 엔드포인트 리팩토링 | 요약 완료 |

---
총 2개 티켓 / 3개 레포 / 4개 세션 / 1개 미정리
```

#### B. 특정 티켓 상세 (티켓 번호 지정)

해당 티켓의 컨텍스트 캐시 요약 + 각 세션 파일의 제목(첫 줄 `# ...`)과 타임스탬프를 보여준다.

```
## PROJ-123 세션 메모리 상세

### 컨텍스트 캐시
- 상태: ✓ 있음 (캐시 일시: 2026-04-03 14:00:00)
- 제목: <Jira 티켓 summary>

### 세션 기록

| # | 레포 | 일시 | 제목 | 상태 |
|---|------|------|------|------|
| 1 | repo-a | 2026-04-03 16:15 | master 머지 충돌 해결 | 요약 완료 |
| 2 | repo-a | 2026-04-06 09:43 | 포인트 할인 UI 구현 | 요약 완료 |
| 3 | repo-b | 2026-04-06 11:00 | — | 미정리 (raw) |
```

- 제목: `.md` 파일의 첫 번째 `# ` 줄에서 `세션 메모리 — ` 이후 부분을 추출한다.
- `*-raw.jsonl` 파일은 상태를 "미정리 (raw)"로 표시하고 제목은 "—"로 표시한다.

## 제약사항

1. **읽기 전용 스킬이다.** 어떤 파일도 수정하거나 삭제하지 않는다.
2. **파일 내용은 제목(첫 줄)만 읽는다.** 전체 내용을 출력하지 않는다.
3. **타임스탬프는 파일명에서 파싱한다.** 파일명 형식: `YYYY-MM-DD-HH_MM_SS.md`
